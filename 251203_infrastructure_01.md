#### AWS 클라우드 컴퓨팅 개념

클라우드 컴퓨팅이란?   
> 인터넷으로 컴퓨터 자원을 빌려 쓰는 기술, 서버나 저장 공간, 프로그램을 필요할 때 빌려씀

클라우드 장점 5가지   
1. 비용 절감
2. 쉬운 확장
3. 빠른 속도
4. 보안
5. 장소에 상관없음

클라우드의 종류
1. 퍼블릭 클라우드: 여러 사람이 함께 공유하며 사용,
2. 프라이빗 클라우드: 나만 접근 가능, ex) 사내 네트워트 서버, = 온프레미스
43. 하이브리드 클라우드: 퍼블릭 + 프라이빗

aws 글로벌 서비스   
aws 리전, 가용영역 엣지 로케이션으로 나눠짐
- 리전: 국가 단위, 서울 도쿄 뉴욕 등등 aws 센서가 모여있는 큰 지역
- 가용역역: 도시 단위, 리전 안에 있는 작은 데이터 센터, 한 리전에 보통 2~3개 위치
- 엣지 로케이션: 센터 구축이 어려운 곳에 설치, cloud Front가 데이터를 빠르게 전달하기 위해 사용하는 작은 저장소

aws  
콘솔, CLI, SDK 로 aws 제어 가능

클라우드 배포 모델
퍼블릭, 하이브리드, 프라이빗

클라우드 컴퓨팅 모델
온프레미스

IaaS(infrastructure as a service)   
> 물리적인 부분은 공급자(AWS사용이면 AWS)가 관리, 사용자는 운영체제, 데이터 스토리지, 애플리케이션 관리

AWS에서는 EC2가 대표적인 서비스   
사용자 관리: 
- 애플리케이션
- 데이터 스토리지
- 런타임
- 미들웨어 
- O/S

공급자 관리:
- 가상화
- 서버
- 스토리지
- 네트워킹

PaaS(platform as a service)   
> 물리적인 부분, OS 등 공급자가 모든 플랫폼을 관리하고 사용자는 데이터와 애플리케이션만 관리, 사용자는 애플리케이션 개발에만 집중할 수 있음

AWS Elastic Beanstalk이 대표적인 서비스   
사용자 관리:    
- 애플리케이션
- 데이터 스토리지

공급자 관리:
- 런타임
- 미들웨어
- O/S
- 가상화
- 서버
- 스토리지
- 네트워킹 

SaaS(software as a service)   
> 모든 부분을 공급자가 관리, 사용자는 서비스를 사용만 하면 됨

공급자 관리(gmail 같은 서비스들)   
애플리케이션, 데이터 스토리지, 런타임, 미들웨어, O/S, 가상화, 서버, 스토리지, 네트워킹 

IAM   
리전이랑 상관없는(글로벌) 계정 관리 서비스   
사용자, 그룹, 역할, 정책으로 구성   
AWS 서비스와 리소스에 대한 액세스 관리  

**계정 보안 강화**
- 루트 계정은 사용자 계정 생성 이후 가능하면 사용하지 않을 것을 권고함
- 사용자 계정 생성 후 필요한 최소 권한만 부여
- 루트 계정과 개별 사용자 계정에 강력한 암호 정책과 mfa(멀티팩터 인증) 적용
- 암호 복잡성(대소문자, 특수문자, 숫자) 요구, 의무 교체 주기를 정의
- 액세스 키 공유 금지   

IAM 사용자 생성   
그룹에 사용자 추가    
ROLE 생성해서 권한 주고 그룹에 사용자를 추가하여 쉽게 권한을 줄 수 있음

IAM 액세스 관리         
사용자 - 루트의 서브 계정   
그룹 - 부서?와 같은 의미   
역할 - 서비스와 서비스간의 권한, AWS 리소스에서 사용하는 자격증명   
정책 - 권한, 콘솔에서 체크해서 권한을 줄 수 도 있고 json으로 설정할 수도 있음

IAM 정책   
AWS 리소스에 대한 액세스 권한을 정의한 것   
사용자, 그룹, 역할에 정책을 연결하여 사용   
정책을 개별 사용자에게 직접 부여 말고 사용자가 속한 그룹에 권한 부여를 권장
